# CS-255
CS-255 Systems Analysis and Design Journal
Overview

This repository includes the business requirements and system design documents for the DriverPass project, demonstrating my ability to analyze requirements and design a functional system.

Summary

The system was designed to help users prepare for driving tests by managing accounts, scheduling sessions, and tracking progress in a reliable, user-friendly way.

Strengths

I clearly defined technical requirements and used UML diagrams to show system structure, interactions, and key processes.

Improvements

I would add more detail to UML diagrams and better account for edge cases and user scenarios.

User Needs

I focused on usability, security (authentication/encryption), and reliability, ensuring the system meets user expectations and protects data.

Approach

I identify requirements, model the system with UML, and define technical components like servers, databases, and security, with a focus on scalability.

Technical Overview
Server-based system with database support
Handles user input, authentication, and transactions
Secure and reliable for multiple users
Reflection

This project improved my ability to connect user needs with system design and reinforced the value of clear planning and documentation.
